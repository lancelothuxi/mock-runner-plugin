Project Overview

This document provides technical specifications and guidelines for the Mock Runner Plugin project.

Key Features

  • Runtime method mocking using Java Agent
  • Support for Dubbo and Feign interfaces
  • Real-time configuration updates
  • Kubernetes integration with sidecar pattern
  • Comprehensive logging and monitoring

Architecture

The system follows a modular architecture with clear separation of concerns:

  1. Agent Layer: ByteBuddy-based method interception
  2. Configuration Layer: JSON-based mock rule management
  3. Integration Layer: Framework-specific interceptors
  4. Monitoring Layer: Real-time logging and metrics
